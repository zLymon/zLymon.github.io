关于this指向的问题，可以简单地总结成下面七点:

# 1. 在全局上下文中使用this

在全局上下文中默认this指向window/global，严格模式下指向undefined。

console.log(this) // window
1

# 2. 直接调用函数

直接调用函数的情况下，this的指向和情况1相同。

// demo1
function f1 () {
  console.log(this) // window
  function f2 () {
    console.log(this) // window
  }
}

// demo2
let obj = {
  a: function () {
    console.log(this)
  }
}

let func2 = obj.a
func() // window

// demo3
let obj2 = {
  a: function () {
    function f () {
      console.log(this)
    }
    // 即使是通过对象调用了方法a，从而调用了方法f
    // 但调用的形式仍然是直接调用，所以this指向window
    f() // window
  }
}
obj2.a()

# 3. 对象.方法的形式调用

以对象.方法形式调用的函数中的this，指向这个对象。它和情况2中的demo2区别在于，这里用对象obj调用了方法a，而情况2中的demo3则是将方法a赋值给了另一个变量。

let obj = {
  a: function () {
    console.log(this)
  }
}
obj.a() // obj

# 4. DOM事件绑定

onclick和addEventListener事件中，this默认指向绑定事件的元素，IE则是使用attchEvent，this默认指向window。

// IE
const elem = document.getElementByTagName('div')[0]
elem.attachEvent('onclick', function () {
  console.log(this) // window
})

// 非IE
elem.addEventListener('click', function () {
  console.log(this) // elem
})
document.getElementsByTagName('button')[0].onclick = function () {
  console.log(this) // button
}

# 5. new构造函数绑定

构造函数中的this指向实例对象。

function Parent (name, age) {
  this.name = name
  this.age = age
}
const child = new Parent('bob', 18)
console.log(child.name) // 'bob'
console.log(child.age) // 18

# 6. 箭头函数

箭头函数中是没有this的，也不能够绑定，但是箭头函数中的this会指向当前最近的非箭头函数的this，如果找不到最近的非箭头函数的this，则该this会指向window。

let obj = {
  a: function () {
    let do = () => {
      console.log(this)
    }
    do()
  }
}
obj.a() // 找到最近的非箭头函数a，a绑定着obj，所以this指向了obj

# 7. call/apply/bind

这三个方法中，this指向这三个方法所绑定的对象。

function f1 () {
  console.log(this)
}

f1() // window

let obj = {}

f1.call(obj) // obj
// apply、bind同理
