js中判断相等有两种方法，一个是==操作符，还有一个是===操作符，这两个有什么区别呢？

可以这么认为:

  • ===是严格相等，左右两边不仅值要相等，类型也要相等。
  • ==是抽象相等，只要值相等，就会返回true，但是在比较的过程中会涉及到类型转换。

# 严格相等 ===

先看===内部是如何进行比较的，先来看ECMA上的定义

The comparison x === y, where x and y are values, produces true or false. Sucna comparison is performed as follows:

    1. If Type(x) is different from Type(y), return false.
    1. If Type(x) is Number, then
    • 2.1. If x is NaN, return false.
    • 2.2. If y is NaN, return false.
    • 2.3. If x is the same Number value as y, return true.
    • 2.4. If x is +0 and y is -0, return true.
    • 2.5. If x is -0 and y is +0, return true.
    • 2.6. Return false.
  1. Return SameValueNonNumber(x, y).

NOTE This algorithm differs from the SameValue Algorithm in its treatment of signed zeroes and NaNs.

    1. 如果x和y的类型不同，返回false
    1. 如果x为Number类型，那么：
    • 2.1. 如果x为NaN，返回false
    • 2.2. 如果y为NaN，返回false
    • 2.3. 如果x和y的数值相等，返回true
    • 2.4. 如果x为+0并且y为-0，返回true
    • 2.5. 如果x为-0并且y为+0，返回true
    • 2.6. 其余的返回false

注意： 该算法和同值算法的区别在于对有符号的0和NaN的处理。

通过ECMA的定义可以发现，===与我们所认知的是一样的，类型与值同时相等才返回true，但是要注意的是，它并不能正确的判断+0和-0，还有NaN的相等。那么要怎么正确判断+0和-0，还有NaN的比较呢，可以通过Object.is来判断。

Object.is(NaN, NaN) // true
Object.is(+0, -0) // false

# 抽象相等 ==

同样，我们先看ECMA上对==的定义是怎样的

The comparison x == y, where x and y are values, produces true of false, Such a comparison is performed as follows:

    1. If Type(x) is the same as Type(y), then
    • 1.1 Return the result of performing Strict Equality Comparison x === y.
    1. If x is null and y is undefined, return true.
    1. If x is undefined and y is null, return ture.
    1. If Type(x) is Number and Type(y) is String, return the result of the comparison x === ToNumber(y).
    1. If Type(x) is String and Type(y) is Number, return the result of the comparison ToNumber(x) == y.
    1. If Type(x) is Boolean, return the result of the comparison ToNumber(x) == y.
    1. If Type(y) is Boolean, return the result of the comparison x == ToNumber(y).
    1. If Type(x) is either String, Number, or Symbol and Type(y) is Object, return the result of the comparison x == ToPrimitive(y).
    1. If Type(x) is Object and Type(y) is either String, Number, or Symbol, return the result of the comparison ToPrimitive(x) == y.
    1. Return false.
  1. 如果x和y的类型相同，则返回x === y的结果
  2. 如果x为null并且y为undefined，返回true.
  3. 如果x为undefined并且y为null，返回true.
  4. 如果x为Number类型并且y为String类型，返回x == ToNumber(y)的结果.
  5. 如果x为String类型并且y为Number类型，返回ToNumber(x) == y的结果.
  6. 如果x为Boolean类型，返回ToNumber(x) == y的结果.
  7. 如果y为Boolean类型，返回x == ToNumber(y)的结果.
  8. 如果x为String、Number或Symbol类型的其中一种，并且y为Object类型，返回x == ToPrimitive(y)的结果.
  9. 如果x为Object类型并且y为String、Number或Symbol类型的其中一种，返回ToPrimitive(x) == y的结果.
  10. 其余的返回false.

可以看到==的定义极其的复杂。。我们先对上述的定义进行整理，可以得到这样简化后的定义：

  1. 同时存在null和undefined的情况下，都返回true.
  2. 同时存在Number和String的情况下，会先将String类型用ToNumber操作转换类型后再次比较.
  3. 有Boolean类型的情况下，先将Boolean类型用ToNumber操作转换类型后再次比较.
  4. 若一方为Object类型，另一方为String、Number或Symbol类型的其中一种，先将Object类型使用ToPrimitive操作转换类型后再次比较.

通过简化后的定义，我们发现这几个重要操作：

  • ToNumber()
  • ToPrimitive()

# ToNumber()

在ECMA上各个类型使用ToNumber返回的值如下表

初始类型 返回结果
Undefined NaN
Null +0
Boolean 初始值为true返回1，false返回+0
Number 返回初始值
String 特殊转换规则
Symbol 抛出TypeError错误
Object 1. Let primValue be? ToPrimitive(argument, hint Number). 2.Return ? ToNumber(primValue).

ToNumber中的String和Object类型没看懂具体的转换规则是怎样的😭

# ToPrimitive()

这个在ECMA上的定义也很复杂，我在一篇文章上看到了对它的一个总结：

  1. ToPrimitive(input, hint)转换为原始类型的方法，根据hint目标类型进行转换。
  2. hint只有两个值：String和Number
  3. 如果没有传入hint，Date类型的input的hint默认为String，其他类型的input的hint默认为Number
  4. Number类型先判断valueOf()方法的返回值，如果不是，再判断toString()方法的返回值
  5. String类型先判断toString()方法的返回值，如果不是，再判断valueOf()方法的返回值

# 题目

根据上面的知识点，来尝试解答一道题： 为什么[] == ![]返回true？

  1. 一元运算符优先级比相等操作符的优先级高，所以先计算![]，得到的结果为false [] == false

  2. 一边有Boolean类型的时候，先将Boolean转成Number类型，false转成Number得到结果为0 [] == 0

  3. 当一边为Object类型，另一边为Number类型时，先根据ToPrimitive()将Object类型转为原始值，先执行valueOf方法，[].valueOf得到的结果还是 [],再执行toString方法，得到'' '' == 0

  4. 当一边为String，另一边为Number时，将String转为Number，''字符串转为Number得到0 0 == 0

  5. 有上面过程可得出[] == ![]

# 扩展

从上面对ToPrimitive的定义我们可以知道，Object转为原始值，是先调用valueOf()方法，但是在es6中多了symbol类型，在Object类型有Symbol.toPrimitive()方法的情况下，会先调用该方法，代码如下

const obj = {
  value: 3,
  valueOf () {
    return 4
  },
  toString () {
    return '5'
  },
  [Symbol.toPrimitive] () {
    return 6
  }
}
console.log(obj + 1) // 输出7
