这道题来源于前几天群里有人讨论一道面试题：如何实现子元素宽高均为父元素宽度的一半？

首先宽度很好实现，子元素的宽度为50%即可；但是高度也需要是父元素宽度的一半要怎么实现呢？

经过大佬们的讨论得出有下面两种实现方法：

  1. js实现的思路很简单：用js获取父元素的宽度，然后将子元素的高度设置为父元素的宽度的50%即可，就不举例了。
  2. css实现的话，可以将子元素的padding-bottom属性设置为50%即可。原理如下

在w3school上我们可以看到这样一个定义：

值 描述
% 定义基于父元素宽度的百分比下内边距。此值不会如预期地那样工作与所有的浏览器中。

所以当我们将子元素的padding-bottom设置为50%时，其高度自然就是父元素的宽度的一半了。

下面为实现的demo

<div class="container">
  <div 
    class="parent" 
    :style="`width: ${parentWidth}px;height: 200px`">
    <div class="child"></div>
  </div>
  <label>在此输入父元素的宽值以查看变化：</label>
  <input type="number" v-model="parentWidth">
</div>
<script>
export default {
  data () {
    return {
      parentWidth: 200
    }
  }
}
</script>
<style>
.parent {
  width: 200px;
  height: 200px;
  border: 1px solid red;
}
.child {
  width: 50%;
  padding-bottom: 50%;
  border: 1px solid blue;
}
</style>

打开开发者工具检查元素，我们可以发现子元素其实本身的高度是为0的，只是被padding-bottom填充了其高度。

虽然说日常开发中可能不会有这样的需求，但是这道面试题可能考察的就是我们在学习或查看文档过程中是否有仔细看api的定义。
