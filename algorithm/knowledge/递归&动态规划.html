# 递归&动态规划

# 递归

递归的概念可以参考该回答：递归概念

以递归最经典的一个题目为例：斐波那契数列

斐波那契数列指的是这样一个数列：1, 1, 2, 3, 5, 8, 13, 21, 34...

可以看出该数列除了前两项以外，每一项的值都是前两项之和，由此可以得出这么一个公式: F(n) = F(n-1) + F(n-2)

现在如果要求出该数列第20项的值，就可以用递归来实现

function fibo (n) {
  if (n === 1 || n === 2) {
    return 1
  }

  return fibo(n - 1) + fibo(n - 2)
}
fibo(20) // 6765

因为我们已经知道了该数列前几项的值，并且该数列的规律又是当前值为前两项的和，所以我们可以将每次计算都丢给前两项的计算，直到有值的情况下再返回。但是递归的效率特别低，例如说当n=5时，要先求fibo(4)和fibo(3)的值，然后再往下求值。 可以发现几乎每个值都被重复计算了，如fibo(3)。当n的数值特别大时，重复计算的值会更多，所以说递归的效率是比较低的。

那么该如何优化这个递归呢，既然他会重复求值，我们可以将得到的值都记录下来，每次求值时，若存在记录的话，直接取值即可。这个方法被称作记忆化递归

# 记忆化递归

既然是记忆化递归，我们则需要一个数组来存储每次计算出来的值，在递归过程中若遇到计算过的值，则直接取出该值即可。

function fibo (n) {
  if (n === 0) return 0
  let memo = Array(n + 1).fill(0)
  return helper(memo, n)
}
function helper (memo, n) {
  if (n === 1 || n === 2) return 1
  if (memo[n] !== 0) return memo[n]
  memo[n] = helper(memo, n-1) + helper(memo, n-2)
  return memo[n]
}

虽然记忆化递归提升了很高的效率，但是本质上他还是递归，那么有没有办法可以通过循环来解决呢？我们可以通过动态规划来解决。

# 动态规划

关于动态规划的概念，可以参考该回答：动态规划概念

还是斐波那契数列这个例子，我们前面用递归都是直接将要求的值n开始一步步往下求值的，直到获得确切的值之后才返回。而动态规划则是自底向上计算，既然前面的值已知，那么就从前面开始计算，直到计算出n的值。

function fibo (n) {
  let dp = Array(n + 1).fill(0)
  dp[1] = 1
  dp[2] = 1
  for (let i = 3; i <= n; i++) {
    dp[i] = dp[i - 1] + dp[i - 2]
  }

  return dp[n]
}

这里我们使用dp数组来存储之前计算过的值，但是我们其实只想得到n对应的值，所以我们通过变量来记录每一步的值，每循环一次，就抛弃之前的值，只记录当前的值为多少即可，这样可以减少对空间的占用。

function fibo (n) {
  if (n === 1 || n === 2) {
    return 1
  }

  let prev = 1, next = 2, temp = 0

  for (let i = 3; i < n; i++) {
    temp = prev + next
    prev = next
    next = temp
  }

  return temp
}
